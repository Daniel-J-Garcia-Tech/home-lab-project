## Configuration Commands Used

### VLAN Creation
```
vlan 15
 name SALES
vlan 25
 name ENGINEERING
vlan 88
 name MANAGEMENT
```

### Port Assignment
```
interface range fastethernet 0/1-2
 switchport mode access
 switchport access vlan 15

interface range fastethernet 0/3-4
 switchport mode access
 switchport access vlan 25
```

### Management VLAN
```
interface vlan 1
 no ip address
 shutdown

interface vlan 88
 ip address 10.100.50.5 255.255.255.0
 no shutdown

ip default-gateway 10.100.50.1
```

## Key Differences from Lab 2 Original
- Different VLAN numbers (15, 25, 88 vs 10, 20, 99)
- Different department names (SALES, ENGINEERING vs ACCOUNTING, IT_DEPT)
- 10.x.x.x IP scheme vs 192.168.x.x scheme
- Different timeout value (8 minutes vs 5 minutes)
- Practice with variety builds adaptability

## Real-World Application
- **VLAN 15 (SALES):** Could represent customer-facing systems or CRM workstations
- **VLAN 25 (ENGINEERING):** Could represent development environments or CAD workstations
- **VLAN 88 (MANAGEMENT):** Network device management and monitoring
- Segmentation prevents data leakage between departments
- Critical for compliance (SOX, HIPAA, government security standards)

## Lessons Reinforced

### Consistent Configuration Patterns
- Same security hardening across all switches
- Standardized VLAN numbering makes scaling easier
- Descriptive names aid troubleshooting

### IP Addressing Schemes
- VLAN number can match third octet (VLAN 15 = 10.15.0.0/24)
- Makes IP-to-VLAN mapping intuitive
- Simplifies documentation and troubleshooting

### Security Through Segmentation
- Each department isolated by default
- Lateral movement prevented in case of breach
- Foundation for Zero Trust architecture
